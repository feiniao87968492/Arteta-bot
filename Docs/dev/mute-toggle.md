# 塔闭嘴/塔说话 开关系统

> 开发者文档：理解群聊静音开关机制。

## 1. 背景

允许群成员通过指令控制机器人是否响应 `A/塔子/阿尔特塔` 指令触发。`@机器人` 不受影响。用于群内需要暂时让机器人安静的场景。

## 2. 存储方式

静音状态以 **内存 Dict** 形式存储，**重启后自动恢复说话**。

- 数据结构：`muted_groups: Dict[str, bool]`，key = group_id
- 默认值：`False`（未静音）
- 不写磁盘，不依赖 SQLite 或文件

## 3. 指令

| 指令 | 功能 |
|------|------|
| `塔闭嘴` | 将当前群标记为静音。已静音时回复"我已经闭嘴了，别喊了。" |
| `塔说话` | 清除当前群静音标记。已说话时回复"我不是正在说着吗？" |

### 3.1 塔闭嘴

使用 `mute_cmd` 处理器（优先级 5，阻塞模式）：

1. 从 `muted_groups` 检查当前群是否已静音
2. 已静音 → finish("我已经闭嘴了，别喊了。")
3. 未静音 → 设置 `muted_groups[group_id] = True`，记录日志，finish("好的，我闭嘴了。")

### 3.2 塔说话

使用 `unmute_cmd` 处理器（优先级 5，阻塞模式）：

1. 从 `muted_groups` 检查当前群是否已静音
2. 未静音 → finish("我不是正在说着吗？")
3. 已静音 → `del muted_groups[group_id]` 清除键，记录日志，finish("我回来了。")

## 4. 集成点

- arteta_chat.py 在 `handle_chat_cmd` 入口检查 `is_muted(str(event.group_id))`，已静音则静默返回
- `is_muted(group_id)` 对非群聊事件返回 `False`
- `@机器人` 走 `at_cmd` 路径，不受影响

## 5. 边界情况

| 场景 | 行为 |
|------|------|
| 重复调用"塔闭嘴" | 提示"我已经闭嘴了，别喊了。"，状态不变 |
| 重复调用"塔说话" | 提示"我不是正在说着吗？"，状态不变 |
| 私聊使用 | `GroupMessageEvent` 不匹配，指令处理器不触发 |
| 多群 | 每个群独立状态 |
| 重启 | 清空内存，自动恢复说话 |
| 其他指令 | 不阻塞，正常使用 |

## 6. 相关文件

| 文件 | 说明 |
|------|------|
| `plugins/arteta_mute.py` | 开关系统的全部实现（约 38 行），包括指令注册、状态管理、导出函数 |
| `plugins/arteta_chat.py` | 在 `handle_chat_cmd` 入口引入 mute 检查（`is_muted`） |
